# Спецификация — SPEC-001

**Дата:** 17.06.2026
**Статус:** Утверждено

## Обзор

Пакет доработок к менеджеру задач v1.0. Расширяет работу с задачами (редактирование, статус «сделано», ручной порядок, пометки «важное»/«дедлайн»), вводит визуальную организацию доски (значки, деление проектов на локальные/глобальные) и закреплённую зону наверху. Закреплённая зона объединяет панель «Дела на день» (три слота, выделена жёлтым) и проекты, которые пользователь закрепил; ниже неё — обычная доска. Почти всё — на вебе; бот затрагивается минимально (только отображение значков). Базовая модель v1.0 (проекты → задачи) сохраняется.

## Фичи

### Фича 1: Значки к проектам

**Описание:** У каждого проекта может быть значок (эмодзи). Задаётся при создании/редактировании проекта на вебе, необязателен. Отображается в вебе рядом с названием проекта и в боте (в кнопках выбора проекта и в заголовках групп `/tasks`).

**Сценарии:**
1. Пользователь создаёт/редактирует проект → выбирает эмодзи из пикера → значок сохраняется и показывается рядом с названием.
2. Пользователь убирает значок → проект отображается без значка.
3. Пользователь открывает бота → видит значок проекта в кнопках и в выводе `/tasks`.

**Граничные случаи:**
- Если значок не задан → проект отображается без значка, выравнивание названий не ломается.
- Если выбран многосоставный эмодзи (флаг, семья, модификатор тона) → сохраняется и рендерится как один символ; не обрезается.
- Если значок не поддерживается шрифтом/клиентом бота → показывается как есть (fallback платформы), задача не падает.
- Значок задаётся только на вебе; через бота установить значок нельзя (бот его только показывает).

**Критерии приёмки:**
- [ ] У проекта можно задать и убрать значок-эмодзи на вебе.
- [ ] Значок виден в вебе рядом с названием проекта.
- [ ] Значок виден в боте: в кнопках выбора проекта и в заголовках групп `/tasks`.
- [ ] Проект без значка отображается корректно и в вебе, и в боте.

### Фича 2: Редактирование задач

**Описание:** На вебе можно отредактировать текст существующей задачи. Перенос задачи в другой проект в эту фичу не входит.

**Сценарии:**
1. Пользователь нажимает «редактировать» у задачи → меняет текст → сохраняет → текст обновлён на месте.
2. Пользователь начинает редактирование → отменяет → текст остаётся прежним.

**Граничные случаи:**
- Если новый текст пустой или только из пробелов → сохранение отклоняется, показывается ошибка, старый текст сохраняется.
- Если в начале/конце текста есть лишние пробелы → обрезаются перед сохранением.
- Если текст очень длинный → применяется тот же лимит длины, что при создании задачи (единое правило).
- Если задача была удалена из этого же проекта через бота, пока открыта форма редактирования → при сохранении показывается понятная ошибка «задача больше не существует», изменения не применяются.

**Критерии приёмки:**
- [ ] Текст задачи можно изменить и сохранить на вебе, изменение видно сразу.
- [ ] Пустой текст и текст из одних пробелов не сохраняются.
- [ ] Отмена редактирования не меняет задачу.
- [ ] Сохранение в уже удалённую задачу не создаёт «призрак» и даёт внятную ошибку.

### Фича 3: Пометки задач — «важное» и «дедлайн»

**Описание:** У задачи есть два независимых атрибута: флаг «важное» (вкл/выкл, визуальная подсветка) и «дедлайн» (необязательная дата). Дедлайн на этом этапе только отображается — он не добавляет задачу в панель «Дела на день» автоматически.

**Сценарии:**
1. Пользователь включает «важное» у задачи → задача визуально выделяется. Выключает → выделение снимается.
2. Пользователь задаёт дедлайн (дату) → дата показывается у задачи.
3. Наступает/проходит дата дедлайна → задача помечается как просроченная (визуальный акцент).
4. Пользователь убирает дедлайн → дата и пометка просрочки исчезают.

**Граничные случаи:**
- Если дедлайн задан на прошедшую дату → разрешено, задача сразу показывается просроченной.
- Если задача с дедлайном помечена «сделано» (Фича 4) → акцент просрочки снимается (выполненное не считается просроченным).
- Если у задачи одновременно «важное» и просроченный дедлайн → оба индикатора видны и не перекрывают друг друга.
- «Важное» и «дедлайн» независимы: любой может быть включён без другого.

**Критерии приёмки:**
- [ ] Флаг «важное» включается/выключается и даёт видимую подсветку.
- [ ] Дедлайн (дата) задаётся, отображается и убирается.
- [ ] Просроченная по дате задача визуально отличается.
- [ ] Выполненная задача не показывается просроченной.
- [ ] Дедлайн не добавляет задачу в панель «Дела на день» сам по себе.

### Фича 4: Зачёркивание задач (статус «сделано»)

**Описание:** На вебе задачу можно пометить как выполненную — она зачёркивается, но остаётся в проекте на своём месте. Пометку можно снять. Полное удаление задачи сохраняется как отдельное действие. Бот статус «сделано» не отображает и не меняет.

**Сценарии:**
1. Пользователь нажимает на задачу (или её отметку) → задача зачёркивается, остаётся в проекте.
2. Пользователь снимает пометку → зачёркивание убирается, задача снова обычная.
3. Пользователь удаляет задачу (как в v1.0) → задача исчезает полностью.

**Граничные случаи:**
- Если зачёркнутую задачу удаляют → удаляется полностью, как любую другую.
- Если зачёркнутую задачу редактируют (Фича 2) → текст меняется, статус «сделано» сохраняется.
- Если зачёркнутая задача добавлена в слот панели (Фича 8) → в слоте она показывается зачёркнутой.
- Зачёркивание не меняет позицию задачи в проекте (порядок из Фичи 5 сохраняется); задача не уезжает вниз автоматически.
- В боте (`/tasks`) зачёркнутая задача выводится как обычная, без отметки выполнения (статус — веб-концепция).

**Критерии приёмки:**
- [ ] Задачу можно пометить «сделано» и снять пометку на вебе.
- [ ] Зачёркнутая задача остаётся в проекте на своей позиции.
- [ ] Полное удаление задачи по-прежнему доступно.
- [ ] Статус «сделано» сохраняется при редактировании текста.
- [ ] Бот не показывает и не меняет статус «сделано».

### Фича 5: Перетаскивание задач (ручной порядок)

**Описание:** На вебе задачи внутри проекта можно перетаскивать выше/ниже, задавая ручной порядок. Порядок сохраняется. Перетаскивание между разными проектами в эту фичу не входит. Работает на компьютере и на телефоне (адаптив из v1.0).

**Сценарии:**
1. Пользователь перетаскивает задачу вверх/вниз внутри проекта → порядок меняется и сохраняется, остаётся после перезагрузки.
2. Пользователь добавляет новую задачу → она встаёт в конец списка проекта.

**Граничные случаи:**
- Если во время перетаскивания в этот же проект добавляется задача через бота → новая задача встаёт в конец, перетаскиваемая сохраняет целевую позицию; порядок не ломается.
- Если перетаскиваемую задачу в этот момент удаляют → перетаскивание корректно отменяется, остальной порядок сохраняется.
- Зачёркнутые задачи (Фича 4) тоже можно перетаскивать.
- На телефоне перетаскивание работает через касание (touch), а не только мышью.
- Перетаскивание ограничено пределами одного проекта — задачу нельзя «уронить» в другой проект.

**Критерии приёмки:**
- [ ] Задачи внутри проекта переупорядочиваются перетаскиванием.
- [ ] Новый порядок сохраняется и переживает перезагрузку страницы.
- [ ] Новые задачи добавляются в конец списка.
- [ ] Перетаскивание работает на телефоне (touch) и на компьютере.
- [ ] Задачу нельзя перетащить за пределы её проекта.

### Фича 6: Организация доски — закреплённая зона и локальные/глобальные проекты

**Описание:** Чисто визуальная организация доски на вебе. Доска сверху вниз делится на две области:
1. **Закреплённая зона** — выделенный блок наверху. Всегда содержит панель «Дела на день» (Фича 8) как первый элемент (колонка слева) и проекты, которые пользователь закрепил (до 3), располагающиеся рядом.
2. **Обычная доска** — ниже зоны. Незакреплённые проекты, разделённые по типу: сначала «локальные», затем «глобальные».

У каждого проекта есть тип «локальный»/«глобальный» и признак закрепления. Закреплённый проект показывается в закреплённой зоне, но сохраняет свой тип и возвращается в обычную доску (в свою часть по типу) при откреплении. Панель «Дела на день» закреплена всегда — её нельзя открепить. Тип и закрепление на бота не влияют.

**Сценарии:**
1. Пользователь открывает веб → видит наверху закреплённую зону (панель «Дела на день» + закреплённые проекты), ниже — обычную доску с локальными, затем глобальными проектами.
2. Пользователь закрепляет проект → проект перемещается в закреплённую зону, рядом с панелью.
3. Пользователь открепляет проект → проект возвращается в обычную доску, в свою часть по типу.
4. Пользователь задаёт/меняет тип проекта «локальный»↔«глобальный» → в обычной доске проект показывается в соответствующей части.

**Граничные случаи:**
- Если закреплённых проектов нет → закреплённая зона всё равно показывается и содержит только панель «Дела на день».
- Если уже закреплено 3 проекта и пользователь пытается закрепить 4-й → действие отклоняется с понятным сообщением (сначала открепить один).
- Если закреплённый проект меняет тип → остаётся в закреплённой зоне (закрепление приоритетнее), новый тип запоминается для момента открепления.
- Если в обычной доске нет локальных или нет глобальных проектов → пустая часть не показывается, доска не выглядит сломанной.
- Тип проекта по умолчанию при создании — «локальный»; новый проект создаётся незакреплённым (попадает в обычную доску).
- Порядок: в обычной доске внутри «локальных» и «глобальных» — как в v1.0; среди закреплённых проектов в зоне — в порядке закрепления; панель «Дела на день» всегда первая (слева) в зоне.
- Удаление закреплённого проекта → освобождает один из трёх слотов закрепления; панель остаётся на месте.
- На телефоне закреплённая зона сворачивается в столбик: панель «Дела на день» сверху, под ней закреплённые проекты, затем обычная доска.

**Критерии приёмки:**
- [ ] Наверху доски есть закреплённая зона, всегда содержащая панель «Дела на день».
- [ ] В зону можно вынести до 3 закреплённых проектов; попытка закрепить 4-й отклоняется.
- [ ] Панель «Дела на день» нельзя открепить.
- [ ] Незакреплённые проекты показываются ниже зоны, разделённые на локальные → глобальные.
- [ ] Открепление возвращает проект в обычную доску по его типу.
- [ ] Закрепление и тип не влияют на поведение бота.
- [ ] На телефоне зона корректно сворачивается в столбик.

### Фича 7: Дата и время в шапке

**Описание:** В правом верхнем углу веб-интерфейса показываются текущие дата и время, живые (обновляются сами), по часовому поясу и локали пользователя (из браузера).

**Сценарии:**
1. Пользователь открывает веб → видит справа сверху текущую дату и время.
2. Время идёт → отображение обновляется без перезагрузки страницы.

**Граничные случаи:**
- Часовой пояс берётся из браузера пользователя; сервер на отображение не влияет.
- При смене даты в полночь → дата в шапке обновляется автоматически.
- На узком экране (телефон) → дата/время не наезжают на другие элементы шапки (адаптив).

**Критерии приёмки:**
- [ ] Дата и время видны справа сверху на вебе.
- [ ] Отображение обновляется само, без перезагрузки.
- [ ] Используются локаль и часовой пояс пользователя.
- [ ] На телефоне блок не ломает шапку.

### Фича 8: Панель «Дела на день»

**Описание:** Панель из трёх неименованных слотов-контейнеров, постоянно закреплённая в верхней зоне доски (Фича 6) как её первый элемент — колонка слева, слоты в столбик. Панель визуально выделена мягким тёплым жёлтым фоном, чтобы отличаться от проектов. Пользователь вручную раскладывает задачи по слотам: открывает слот, отмечает нужные задачи из любых проектов, сохраняет слот. Слоты ссылаются на существующие задачи (не копии). Система ничего не выбирает сама. Набор хранится, пока пользователь сам его не изменит; авто-сброса по времени нет.

**Сценарии:**
1. Пользователь нажимает на слот → входит в режим выбора → отмечает нужные задачи (из любых проектов) → сохраняет → отмеченные задачи появляются в слоте.
2. Пользователь снова открывает слот → снимает/добавляет отметки → сохраняет → состав слота обновляется.
3. Пользователь отменяет выбор вместо сохранения → состав слота остаётся прежним.
4. Пользователь смотрит панель → видит во всех трёх слотах текущие задачи на день.

**Граничные случаи:**
- Если задача уже лежит в другом слоте и её отмечают в текущем → она переносится в текущий слот (одна задача — максимум в одном слоте, дублей нет).
- Если задача, лежащая в слоте, удалена из проекта → она исчезает из слота.
- Если задача в слоте отредактирована (Фича 2) → в слоте показывается новый текст.
- Если задача в слоте помечена «сделано» (Фича 4) → в слоте она показывается зачёркнутой, но из слота не пропадает (убрать можно вручную).
- Слот можно сохранить пустым → слот отображается пустым, это допустимо.
- Слотов всегда ровно три, они не именуются, не добавляются и не удаляются.
- Слоты в зоне располагаются вертикально (в столбик), а не в ряд.
- Жёлтым выделена только сама панель «Дела на день»; закреплённые проекты рядом и контейнер зоны остаются в обычной цветовой схеме.
- Панель не сбрасывается сама по времени/в полночь; «новый день» пользователь раскладывает вручную.

**Критерии приёмки:**
- [ ] На вебе есть панель из трёх слотов.
- [ ] Панель постоянно находится в верхней закреплённой зоне, слоты в столбик.
- [ ] Панель имеет мягкий жёлтый фон, отличающий её от проектов.
- [ ] В слот можно добавлять задачи из любых проектов через выбор-отметку с явным сохранением.
- [ ] Одна задача может находиться максимум в одном слоте; повторный выбор переносит её.
- [ ] Слот ссылается на задачу: правки/статус/удаление в проекте отражаются в слоте.
- [ ] Набор слотов сохраняется между сессиями и не сбрасывается автоматически.
- [ ] Отмена выбора не меняет состав слота.

## Технические требования

- **Модель проектов** дополняется полями: значок (эмодзи, необязательное), тип (`local` / `global`, по умолчанию `local`), признак закрепления (булево, одновременно не более 3 закреплённых на пользователя).
- **Модель задач** дополняется полями: порядковый индекс внутри проекта (для ручного порядка), флаг «важное» (булево), дедлайн (дата, необязательная), статус «сделано» (булево).
- **Хранение панели «Дела на день»**: три слота, каждый — упорядоченный набор ссылок на существующие задачи; ссылочная целостность (удаление задачи убирает её из слота). Одна задача — не более чем в одном слоте. Панель рендерится в закреплённой зоне (Фича 6) как постоянный первый элемент.
- **Визуальная организация (веб)**: закреплённая зона — выделенный блок наверху доски, содержащий панель «Дела на день» (мягкий тёплый жёлтый фон, слоты в столбик) и до 3 закреплённых проектов рядом. Ниже — обычная доска (локальные → глобальные). Жёлтый применяется только к панели, не к зоне и не к проектам.
- **Бот**: единственное изменение — отображение значка проекта в кнопках и в `/tasks`. Поля «важное», «дедлайн», «сделано», порядок, тип/закрепление, панель — бот не показывает и не меняет.
- **Совместимость**: существующие проекты и задачи v1.0 должны корректно работать после миграции (значок пуст, тип `local`, не закреплён, «важное» выкл., дедлайн пуст, «сделано» выкл., порядок — текущий/по созданию).
- **Адаптив**: перетаскивание (Фича 5), панель (Фича 8) и шапка с датой (Фича 7) работают на телефоне и на компьютере.

## Зависимости

- Базируется на v1.0 (структура проекты → задачи, веб-авторизация логин/пароль, Telegram-бот, общая SQLite).
- Внутренние зависимости пакета: организация доски (Фича 6) объединяет панель «Дела на день» (Фича 8) и закреплённые проекты в одной верхней зоне; панель опирается на статус «сделано» (Фича 4) и редактирование (Фича 2) для корректного отображения; отрисовка использует значки (Фича 1).
- Идея из бэклога «авто-появление дел по дате дедлайна» в этот пакет **не входит**: дедлайн пока только отображается, панель наполняется вручную.

## Заметки для разработчика

- Принцип «система ничего не решает сама» для панели: никакого авто-наполнения, авто-баланса и авто-сброса. Только ручная раскладка.
- Зачёркивание — это статус, а не удаление; полное удаление остаётся отдельным действием. В v1.0 статуса «сделано» сознательно не было — это намеренное изменение модели данных.
- Деление локальные/глобальные — чисто визуальное (порядок в обычной доске), применяется только к незакреплённым проектам ниже закреплённой зоны; никакой логики доступа или поведения.
- Закрепление: панель «Дела на день» закреплена всегда и не откручивается; проекты пользователь закрепляет сам (до 3). Закреплённый проект остаётся в зоне независимо от типа, тип сохраняется для момента открепления.
- Изначальная идея 4-го слота «творчество/спорт» и авто-«баланса» в этот пакет не включена (по решению на этапе проектирования). При необходимости добавляется отдельно: 4-й слот + ручная пометка задач как «творчество/спорт».
- Версионирование по правилам v1.0: этот пакет — новая функциональность, значит `2.0`; не забыть changelog в README.
