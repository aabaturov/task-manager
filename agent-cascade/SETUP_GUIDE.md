# Настройка системы агентов для CRM Bot

## Что здесь

Это набор файлов для настройки системы из трёх агентов в Claude Desktop:

```
CLAUDE.md                              — Общие инструкции проекта
.claude/agents/developer.md            — Агент-разработчик
.claude/agents/tester.md               — Агент-тестировщик
.claude/commands/run-pipeline.md       — Команда запуска пайплайна
.agent_docs/templates/SPEC_TEMPLATE.md — Шаблон спецификации
```

## Установка — 3 шага

### Шаг 1. Скопируй файлы в проект

Скопируй содержимое в корень твоего проекта `C:\Else\...\AI projects\crm_bot\`:

```
crm_bot/
├── CLAUDE.md                          ← скопировать (или дополнить существующий)
├── .claude/
│   ├── agents/
│   │   ├── developer.md               ← скопировать
│   │   └── tester.md                  ← скопировать
│   └── commands/
│       └── run-pipeline.md            ← скопировать
├── .agent_docs/
│   ├── specs/                         ← создать пустую папку
│   ├── dev_reports/                   ← создать пустую папку
│   ├── test_reports/                  ← создать пустую папку
│   ├── bugfixes/                      ← создать пустую папку
│   └── templates/
│       └── SPEC_TEMPLATE.md           ← скопировать
├── handlers/
├── services/
├── models/
├── tests/
└── ... остальные файлы проекта
```

Если у тебя уже есть CLAUDE.md — добавь содержимое из моего файла
к существующему, не заменяй полностью.

### Шаг 2. Создай пустые директории

```
mkdir -p .agent_docs/specs
mkdir -p .agent_docs/dev_reports
mkdir -p .agent_docs/test_reports
mkdir -p .agent_docs/bugfixes
```

### Шаг 3. Проверь, что всё работает

Открой Claude Desktop → вкладка Code → укажи папку проекта.
Набери `/` — ты должен увидеть команду `run-pipeline` в списке.
Также набери `/agents` — ты должен увидеть `developer` и `tester`.

## Как пользоваться

### Этап 1 — Аналитика (Chat)

1. Открой Claude Desktop → вкладка **Chat**
2. Обсуди фичи, как обычно
3. Когда готов, скажи: **"Финализируй спецификацию SPEC-004"**
4. Я создам файл `.agent_docs/specs/SPEC-004.md` по шаблону
   со всеми фичами, граничными случаями и критериями приёмки
5. Ты ревьюишь и говоришь "ОК" или просишь правки

### Этап 2 — Запуск разработки (Cowork / Code)

1. Переключись на вкладку **Cowork** (или используй Dispatch)
2. Напиши одну фразу:

   ```
   Запусти разработку и тестирование по спецификации SPEC-004
   ```

   Или используй команду в Code:

   ```
   /run-pipeline SPEC-004
   ```

3. Дальше всё идёт автоматически:
   - Разработчик реализует фичи
   - Тестировщик проверяет
   - Если баги — разработчик фиксит (до 3 итераций)
   - Ты получаешь финальный отчёт

### Этап 3 — Результат

После завершения проверь отчёты:

- `.agent_docs/dev_reports/DEV-004.md` — что сделал разработчик
- `.agent_docs/test_reports/TEST-004.md` — результаты тестирования

Если остались нерешённые проблемы — обсуди их со мной в Chat.

## Важные моменты

**Токены.** Каждая итерация фикс→тест тратит токены.
Лимит в 3 итерации — защита от бесконечных циклов.

**CLAUDE.md.** Если ты изменишь структуру проекта (папки, конвенции) —
обнови CLAUDE.md, чтобы агенты знали о новой структуре.

**Нумерация.** Веди нумерацию спецификаций последовательно:
SPEC-001, SPEC-002, SPEC-003... Это нужно для связывания
документов между собой.

**Агенты — не магия.** Они работают в рамках одной модели
с разными промптами. Если спецификация плохая — результат
будет плохой. Вкладывай время в аналитику, и разработка
пойдёт гладко.

## Известная проблема при установке (и как мы её решили)

Если `/run-pipeline` не появляется или выдаёт "Unknown skill":

1. Файл команды должен лежать в `.claude/commands/run-pipeline.md`
   (а НЕ в `.claude/skills/`).
2. Поле `description` в YAML-заголовке должно быть **в одну строку**,
   а не многострочным через `|`. Многострочный вариант Claude Code
   может не распознать.
3. После правки начни **новую сессию** в Code (не текущую) и
   попробуй `/run-pipeline SPEC-000` снова.
