# Менеджер задач (веб + Telegram-бот)

Персональный менеджер задач для одного пользователя. Две функции:
быстро «выгрузить из головы» задачу и в нужный момент увидеть список актуальных
задач. Сущности — **проекты**, внутри них **задачи**. Без дедлайнов,
приоритетов и статусов: задача либо есть, либо удалена.

**Версия:** см. файл [`VERSION`](VERSION) (сейчас `3.0`).

## Стек

- Backend: FastAPI (Python 3.12) + SQLAlchemy + SQLite
- Frontend: React + Vite (собирается в статику, раздаётся FastAPI)
- Бот: python-telegram-bot 22.x
- Упаковка: Docker + docker-compose, reverse-proxy Caddy (авто-HTTPS)

Веб и бот ходят в одну и ту же базу SQLite через общий volume.

## 1. Локальный запуск (для теста)

Перед заливкой на сервер удобно проверить всё на своём компьютере. Локально
Caddy и домен не нужны — веб поднимается напрямую на `http://localhost:8000`.

1. Установи **Docker Desktop** (Windows/macOS) или Docker Engine (Linux).
2. Создай файл `.env` из примера и заполни его:

   ```bash
   cp .env.example .env
   ```

   Минимум для локального теста:

   - `WEB_LOGIN` / `WEB_PASSWORD` — придумай логин и пароль для входа в веб.
   - `SECRET_KEY` — любая длинная случайная строка
     (`python -c "import secrets; print(secrets.token_hex(32))"`).
   - `TELEGRAM_BOT_TOKEN` и `TELEGRAM_ALLOWED_USER_ID` — нужны, только если хочешь
     проверить бота (как получить — см. ниже). Если бот пока не нужен, можно
     оставить токен пустым, но тогда контейнер `bot` будет падать и
     перезапускаться — это нормально, веб от этого не страдает.
   - `DOMAIN` — для локали можно не трогать.

3. Подними стек **без Caddy** локальным compose-файлом:

   ```bash
   docker compose -f docker-compose.local.yml up --build
   ```

4. Открой веб: <http://localhost:8000> — увидишь страницу логина. Войди под
   `WEB_LOGIN` / `WEB_PASSWORD`, создай проект, добавь задачу.

5. Проверка бота: напиши своему боту в Telegram `/start` — появится постоянная
   клавиатура с кнопками **📋 Все задачи** и **📅 Дела на день**. Пришли любой
   текст — бот предложит выбрать проект; после выбора задача появится и в вебе.
   Кнопка **📋 Все задачи** (или команда `/tasks`) покажет все задачи с
   кнопками удаления, а **📅 Дела на день** — слоты из панели «Дела на день»
   (только просмотр).

Остановить: `Ctrl+C`, затем при желании `docker compose -f docker-compose.local.yml down`.

> База лежит в Docker-volume `db-data`. Чтобы начать с чистого листа:
> `docker compose -f docker-compose.local.yml down -v`.

## 2. Деплой на арендованный VPS

На сервере используется полный `docker-compose.yml`: web + bot + **Caddy**,
который сам получает и продлевает HTTPS-сертификат для твоего домена.

1. **Наведи домен на сервер.** В DNS-настройках домена создай `A`-запись,
   указывающую на публичный IP VPS (например `tasks.example.com → 203.0.113.10`).
   Дождись, пока запись распространится. Открой порты `80` и `443`.

2. **Установи Docker** на сервере (Docker Engine + plugin compose).

3. **Склонируй проект и заполни `.env`:**

   ```bash
   git clone <repo-url> task-manager
   cd task-manager
   cp .env.example .env
   nano .env
   ```

   Обязательно задай:
   - `DOMAIN` — твой домен (тот, что навёл в DNS).
   - `WEB_LOGIN`, `WEB_PASSWORD`, `SECRET_KEY`.
   - `TELEGRAM_BOT_TOKEN`, `TELEGRAM_ALLOWED_USER_ID`.

4. **Запусти стек:**

   ```bash
   docker compose up -d --build
   ```

   Caddy автоматически выпустит TLS-сертификат Let's Encrypt для `DOMAIN` и будет
   проксировать трафик на web-сервис. Через минуту открой `https://<твой-домен>` —
   логин уже по HTTPS, пароль по сети открытым не ходит.

5. Логи и управление:

   ```bash
   docker compose logs -f          # смотреть логи
   docker compose ps               # статус сервисов
   docker compose up -d --build    # обновить после git pull
   docker compose down             # остановить
   ```

> База лежит в volume `db-data` и переживает перезапуски/обновления.

## Как создать бота и узнать свой Telegram ID

1. **Токен бота:** в Telegram напиши [@BotFather](https://t.me/BotFather),
   команда `/newbot`, следуй инструкциям. В конце получишь
   `TELEGRAM_BOT_TOKEN` вида `123456:ABC-DEF...`.
2. **Свой user ID:** напиши боту [@userinfobot](https://t.me/userinfobot) — он
   пришлёт твой числовой `id`. Это значение → `TELEGRAM_ALLOWED_USER_ID`.
   Бот отвечает только этому ID, остальных игнорирует.

## REST API

Все эндпоинты, кроме `/api/login`, требуют авторизации (session-cookie).

| Метод  | Путь                          | Описание                          |
| ------ | ----------------------------- | --------------------------------- |
| POST   | `/api/login`                  | вход (`{login, password}`)        |
| POST   | `/api/logout`                 | выход                             |
| GET    | `/api/projects`               | список проектов                   |
| POST   | `/api/projects`               | создать проект (`{name}`)         |
| DELETE | `/api/projects/{id}`          | удалить проект (с задачами)       |
| GET    | `/api/tasks?project_id=`      | задачи (фильтр по проекту опц.)   |
| POST   | `/api/tasks`                  | создать задачу (`{project_id, text}`) |
| DELETE | `/api/tasks/{id}`             | удалить задачу                    |

## Changelog

### 3.0 (SPEC-002)
Доработка Telegram-бота. Схема БД не меняется.
- **Постоянная клавиатура** — на `/start` бот шлёт reply-клавиатуру с кнопками
  **📋 Все задачи** и **📅 Дела на день**. Произвольный текст по-прежнему
  создаёт задачу через выбор проекта; нажатие кнопки задачу не создаёт.
- **Экран «Дела на день»** (только просмотр) — бот читает слоты из того же
  источника, что веб-панель `DayPanel`: непустые слоты по порядку, заголовок
  `Слот N` (1-based), подпись проекта + текст, выполненные помечены ✅ и
  зачёркиванием. Длинные списки разбиваются по границам слотов.

### 2.0 (SPEC-001)
Пакет доработок до v2.0. Базовая модель (проекты → задачи) сохранена,
существующие v1.0 базы мигрируются автоматически при старте.
- **Значки проектов** — необязательный эмодзи у проекта, виден в вебе и в боте
  (кнопки выбора и заголовки `/tasks`).
- **Редактирование задач** — изменение текста на вебе (пустой текст отклоняется).
- **Пометки** — флаг «важное» (подсветка) и дедлайн (дата); просроченные
  выделяются, выполненные не считаются просроченными.
- **Статус «сделано»** — зачёркивание задачи без удаления; полное удаление
  по-прежнему доступно. Бот статус не показывает.
- **Ручной порядок** — перетаскивание задач внутри проекта (мышь и тач).
- **Организация доски** — закреплённая зона наверху (панель «Дела на день» +
  закреплённые проекты, до 3), ниже — обычная доска (локальные → глобальные).
- **Дата и время в шапке** — живые, по часовому поясу и локали браузера.
- **Панель «Дела на день»** — три слота в столбик, закреплена в верхней зоне
  (мягкий жёлтый фон); ручная раскладка задач из любых проектов; задача
  одновременно максимум в одном слоте.

Тех. детали и критерии приёмки — в `agent-cascade/.agent_docs/specs/SPEC-001.md`,
отчёт о реализации — в `.agent_docs/dev_reports/DEV-001.md`.
