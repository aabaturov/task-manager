# Отчёт разработчика — DEV-001

**Спецификация:** SPEC-001
**Версия:** 2.0
**Статус:** реализовано, бэкенд-автотесты зелёные; ожидается ручная приёмка

## Стек (без изменений)
FastAPI + SQLAlchemy + SQLite (бэкенд), React + Vite (фронтенд),
python-telegram-bot (бот), Docker Compose + Caddy. Реализация строго в рамках
существующего стека и архитектуры v1.0.

## Реализованные фичи

| # | Фича | Где |
|---|------|-----|
| 1 | Значки проектов (эмодзи) | модель, API, бот, веб |
| 2 | Редактирование текста задачи | `PATCH /api/tasks/{id}`, веб |
| 3 | «Важное» + дедлайн (+ просрочка) | модель, API, веб |
| 4 | Статус «сделано» (зачёркивание) | модель, API, веб |
| 5 | Ручной порядок (drag, мышь+тач) | `position`, `POST /reorder`, веб |
| 6 | Закреплённая зона + локальные/глобальные (≤3 закреп.) | модель, `PATCH /projects`, веб |
| 7 | Живые дата/время в шапке | веб (локаль/таймзона браузера) |
| 8 | Панель «Дела на день» (3 слота, в зоне, жёлтая) | таблица `day_slot_items`, API, веб |

> **Уточнённая спека (ревизия SPEC-001):** доска делится на две области —
> **закреплённая зона** наверху (панель «Дела на день» как первый элемент-колонка
> слева + закреплённые проекты рядом) и **обычная доска** ниже (локальные →
> глобальные). Панель «Дела на день» имеет мягкий жёлтый фон (только панель),
> слоты идут **в столбик**, открепить панель нельзя. На телефоне зона
> сворачивается в столбик (панель сверху). Изменения только во фронтенде
> (`Board.jsx`, `DayPanel.jsx`, `styles.css`); бэкенд/модель/бот не менялись.

## Изменения модели данных
- **Project** + `icon` (эмодзи, nullable), `type` (`local`/`global`, default
  `local`), `pinned` (bool), `pinned_at` (для порядка в секции «закреплённые»).
- **Task** + `position` (ручной порядок), `important` (bool), `deadline` (date,
  nullable), `done` (bool).
- **DaySlotItem** — новая таблица: `slot_index` (0..2), `task_id`
  (FK→tasks, ON DELETE CASCADE), `position`; `UNIQUE(task_id)` гарантирует
  «одна задача — максимум в одном слоте».

## Миграция и совместимость
`init_db()` после `create_all` вызывает `_migrate()` — идемпотентно добавляет
недостающие колонки через `ALTER TABLE` и бэкфиллит `position` существующим
задачам по `created_at`. Старые v1.0 базы продолжают работать (значок пуст, тип
`local`, не закреплён, важное/дедлайн/сделано выкл.). Покрыто тестом
`test_migration.py` (на синтетической v1.0-базе в отдельном процессе).

## Обработка граничных случаев (из спеки)
- Значок не задан → выравнивание не ломается; через бота значок не задаётся.
- Пустой/пробельный текст при редактировании → 422, старый текст сохраняется
  (`test_edit_empty_text_rejected`).
- Сохранение в удалённую задачу → 404 «Задача больше не существует».
- Просрочка: прошедшая дата → сразу просрочено; `done` снимает акцент просрочки
  (`isOverdue` учитывает `done`).
- Закрепление 4-го проекта → 409 (`test_pin_limit_three`); тип сохраняется при
  откреплении (`test_pin_keeps_type`).
- Reorder: чужая задача → 400; не упомянутые задачи (добавленные ботом во время
  драга) уходят в конец (`test_reorder_partial_keeps_others_at_end`).
- Слоты: дубли схлопываются, выбор задачи из другого слота переносит её
  (`test_day_slot_set_and_move`), удаление задачи убирает её из слота
  (`test_deleting_task_removes_from_slot`).
- Бот не показывает и не меняет `done`/`important`/`deadline`/порядок/тип/панель.

## Известные ограничения / заметки
- Перетаскивание реализовано на Pointer Events без сторонних библиотек (мышь +
  тач), `touch-action: none` на ручке, оптимистичный порядок с сохранением на
  `pointerup`.
- Дедлайн только отображается и не наполняет панель «Дела на день» автоматически
  (по спеке).
- Фронтенд-сборка (`vite build`) и подъём через Docker выполняются в среде
  пользователя (в моей среде нет Node/Docker) — см. RUN-001.md. Бэкенд-логика
  покрыта автотестами и проверена.
- Суб-агент `tester` в текущей среде как отдельный агент недоступен; проверка
  выполнена через автотесты + ревью кода в рамках этой реализации.
