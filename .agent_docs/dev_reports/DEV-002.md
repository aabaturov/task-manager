## Что реализовано

Доработка Telegram-бота (`backend/bot.py`), python-telegram-bot 22.0. Схема БД
не меняется. Обе фичи из спеки.

### Фича 1: Постоянная reply-клавиатура

- На `/start` бот шлёт приветствие + `ReplyKeyboardMarkup(resize_keyboard=True,
  is_persistent=True)` с двумя кнопками **📋 Все задачи** и **📅 Дела на день**
  в один ряд (`_reply_keyboard()`).
- Подписи кнопок и строки фильтров — единые константы `BTN_TASKS`, `BTN_DAY`
  (эмодзи матчатся посимвольно).
- Кнопка **📋 Все задачи** → `show_all_tasks` → тот же `_send_task_list`, что и
  `/tasks` (идентичный вывод).
- Кнопка **📅 Дела на день** → `day_screen` (Фича 2).
- Порядок регистрации хендлеров строгий: сначала точные обработчики кнопок
  (`MessageHandler(filters.Text([BTN_TASKS]), …)` и `filters.Text([BTN_DAY])`),
  затем общий `filters.TEXT & ~filters.COMMAND` — последним. Нажатие кнопки не
  создаёт задачу.
- Проверка владельца `_is_owner` (`TELEGRAM_ALLOWED_USER_ID`) на всех новых
  точках входа: `start`, `show_all_tasks`, `day_screen`. Не-владелец не получает
  ни клавиатуру, ни данные.
- Повторный `/start` идемпотентен: просто перешлёт клавиатуру, задач не создаёт.

### Фича 2: Экран «Дела на день» (только просмотр)

- `_read_slots(db)` читает слоты из того же источника, что веб (`DaySlotItem`,
  порядок `slot_index, position`), и зеркалит `app.main.get_day_slots`:
  `SLOT_COUNT = 3` слота, каждый — упорядоченный список `task_ids`. Новых
  таблиц/полей нет.
- `_build_day_messages()` формирует сообщения:
  - один блок на каждый **непустой** слот, в порядке `index`;
  - заголовок `Слот N` 1-based (`index + 1`), как в вебе;
  - каждая задача — подпись проекта (`_label`, эмодзи + имя) + текст;
  - выполненная задача: `✅ <s>текст</s>`, невыполненная — без пометки;
  - `parse_mode = HTML`; текст задачи и подпись проекта экранируются через
    `html.escape` (`< > &`), собственная разметка (`<b>`, `<s>`) не ломается;
  - пустые слоты пропускаются (заголовок не выводится);
  - ничего не выбрано → одно сообщение «На день пока ничего не выбрано».
- **Защитный fallback:** висячий `task_id`, не резолвящийся в задачу, молча
  пропускается; если из-за этого слот опустел — слот тоже пропускается; бот не
  падает.
- **Лимит Telegram 4096:** блоки слотов пакуются в сообщения без превышения
  лимита и без разрыва слота посередине (`_build_day_messages` возвращает список
  сообщений, `day_screen` шлёт их по очереди).

## Известные ограничения / трейдофы (из спеки)

- Текст сообщения, точно совпадающий с подписью кнопки (`📋 Все задачи` /
  `📅 Дела на день`), трактуется как нажатие и не может быть сохранён как задача
  — осознанный трейдоф для персонального инструмента (SPEC-002 F1).
- Экран «Дела на день» — только просмотр: кнопок отметки «выполнено» нет.
