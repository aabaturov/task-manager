## Что поднято

Локальный стек через `docker-compose.local.yml` (без Caddy/HTTPS):

- **web** — FastAPI + собранный React, отвечает на `http://localhost:8000`
  (`GET /` → HTTP 200). API под `/api/*`.
- **bot** — Telegram-бот (`python bot.py`), long-polling активен
  (`getUpdates → 200 OK`), 0 рестартов.

> Примечание про бота: при старте однократно мелькнул
> `telegram.error.Conflict: terminated by other getUpdates request` — это
> временный перехват long-polling у другого инстанса с тем же токеном
> (например, бот на VPS). После перехвата локальный бот стабильно получает
> `getUpdates 200 OK`. Если на VPS параллельно работает прод-бот с тем же
> токеном — они будут «отбирать» апдейты друг у друга; для чистой ручной
> проверки на время теста остановите прод-бот.

## Где тестировать

- **Веб:** открой в браузере `http://localhost:8000`
  Логин из `.env`: **admin** / **Aa111111**
- **Бот:** Telegram-бот с токеном из `.env`
  (`@BotFather`-бот, id токена `8876908185:...`). Тестировать только с
  аккаунта-владельца (`TELEGRAM_ALLOWED_USER_ID=528506376`).

## Полезное

```bash
# смотреть логи
docker compose -f docker-compose.local.yml logs -f

# логи только бота / только веба
docker compose -f docker-compose.local.yml logs -f bot
docker compose -f docker-compose.local.yml logs -f web

# статус контейнеров
docker compose -f docker-compose.local.yml ps

# остановить, когда закончишь
docker compose -f docker-compose.local.yml down
```
