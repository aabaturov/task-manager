**Спецификация:** SPEC-001
**Версия:** 2.0
**Статус:** код готов, бэкенд-автотесты зелёные. Сборку фронта и подъём Docker
нужно выполнить в твоей среде (у меня в песочнице нет Node/Docker).

## Как поднять локально

```bash
cd "C:\Else\study\AI projects\Task_manager"
docker compose -f docker-compose.local.yml up --build
```

Открой веб: <http://localhost:8000> (вход — `WEB_LOGIN`/`WEB_PASSWORD` из `.env`).

> Образ пересоберётся (новый фронтенд + бэкенд). База существующая мигрируется
> автоматически — старые проекты/задачи останутся на месте.

### Прогнать бэкенд-автотесты (опционально)
```bash
cd backend
python -m venv .venv && .venv\Scripts\pip install -r requirements-dev.txt
.venv\Scripts\python -m pytest -q      # ожидается 23 passed
```

## Что проверить (по сценариям из спеки)

**Ф1 — значки проектов**
- [ ] Открой настройки проекта (✎) → задай эмодзи → виден рядом с названием.
- [ ] Убери значок (кнопка «нет») → проект без значка, вёрстка не ломается.
- [ ] В Telegram: `/tasks` и выбор проекта при добавлении — значок виден.

**Ф2 — редактирование задач**
- [ ] ✎ или двойной клик по тексту → измени → Enter → текст обновился.
- [ ] Пустой текст → ошибка, старый текст сохранён. Escape — отмена.

**Ф3 — важное / дедлайн**
- [ ] ★ включает «важное» (подсветка), повторно — снимает.
- [ ] 📅 → задай дату; прошедшая дата → «просрочено» (красным).
- [ ] Отметь задачу сделанной → акцент просрочки исчезает.

**Ф4 — статус «сделано»**
- [ ] Кружок слева → задача зачёркивается, остаётся на месте; повторно — снимает.
- [ ] Удаление (×) по-прежнему удаляет полностью.
- [ ] В боте `/tasks` зачёркнутая задача показывается как обычная.

**Ф5 — перетаскивание (порядок)**
- [ ] Тяни за ручку ⋮⋮ вверх/вниз — порядок меняется и сохраняется (F5 — остаётся).
- [ ] На телефоне работает пальцем (touch).
- [ ] Новая задача добавляется в конец.

**Ф6 — организация доски (закреплённая зона)**
- [ ] Наверху есть закреплённая зона: слева панель «Дела на день», рядом —
      закреплённые проекты. Ниже зоны — обычная доска.
- [ ] 📌 закрепляет проект (уезжает в зону, рядом с панелью); открепление
      возвращает в обычную доску по его типу.
- [ ] Закрепи 3, попробуй 4-й → отказ с сообщением. Панель открепить нельзя.
- [ ] В настройках проекта смени тип local/global → в обычной доске проект в
      нужной части (локальные → глобальные).
- [ ] На телефоне зона сворачивается в столбик: панель сверху, под ней проекты.

**Ф7 — дата/время в шапке**
- [ ] Справа сверху идут дата и время, обновляются сами, по твоей локали.
- [ ] На узком экране не ломают шапку.

**Ф8 — панель «Дела на день»**
- [ ] Панель в верхней зоне, слева; мягкий жёлтый фон; три слота **в столбик**.
- [ ] «Изменить» у слота → отметь задачи из разных проектов → «Сохранить».
- [ ] Отметь задачу из слота в другом слоте → она переезжает (не дублируется).
- [ ] Отредактируй/отметь сделанной/удали задачу в проекте → отражается в слоте.
- [ ] «Отмена» не меняет состав слота; панель не сбрасывается сама.

## Полезное
```bash
docker compose -f docker-compose.local.yml logs -f   # логи
docker compose -f docker-compose.local.yml down        # остановить
```

После проверки — дай «ок» (или список замечаний). По инструкции разработчика
коммит делается **только после твоего подтверждения приёмки**; `git push` и
деплой на сервер — отдельной командой.
